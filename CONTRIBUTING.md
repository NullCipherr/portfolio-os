# Contributing

Thank you for your interest in contributing to Portfolio OS.

## Development Setup

1. Fork the repository.
2. Create a feature branch from `main`.
3. Install dependencies with `npm install`.
4. Run the project with `npm run dev`.

## Pull Request Guidelines

- Keep PR scope focused and easy to review.
- Include clear description of behavior changes.
- Update relevant documentation when architecture or behavior changes.
- Ensure `npm run lint` and `npm run build` pass before opening PR.

## Commit Recommendations

Use clear and action-oriented commit messages, for example:

- `feat: add keyboard focus states for taskbar buttons`
- `fix: prevent window overlap on initial open`
- `docs: update operations guide for static deployment`

## Reporting Issues

Please use issue templates and include:

- expected behavior;
- actual behavior;
- reproduction steps;
- environment details.
